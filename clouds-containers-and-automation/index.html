Why Automate?

Automation is about taking manual processes and placing technology around them to make them repeatable.

Automation is the key to speed, consistency, scalability and repeatability.

Why Automate?

  • Scalability
  • Reliability
  • Duplication
  • Consistency
  • Auditability
  • Security

  • Cattle vs pets

Continuous integration

  • Every developer commit triggers a merge/build/test cycle
  • Commits that pass testing are release candidates
  • The process is automated via
    • git hooks
    • jenkins
    • unit tests
    • automated functional / integration tests

Continuous delivery

  • Automated Pipeline from developer commit to production deploy
  • Requires Continuous Integration where every commit triggers all the tests
  • Requires lots of tests
  • Deploying more frequently lowers risk

Continuous delivery antipatterns

  • Deploying software manually
  • Manual configuration of production environments
  • Deploying to a production-like environment only after development is complete

Types of Automation

  • Provisioning
  • Configuration Management
  • Application Deployment
  • Security & Compliance
  • Orchestration
  • Container Orchestration Engines

Provisioning

  • Usually bare metal
  • OS installation
  • Then bootstrap configuration management

Provisioning

  • Red Hat Kickstart
  • FAI
  • Debian Preseed
  • Cobbler

Configuration Management

  • Best known, often equated with "Automation"
  • Mostly about configuring servers (instances)
  • Agentless vs Agents
  • Often declarative syntax

Configuration Management

  • Salt
  • Puppet
  • Chef
  • Ansible

Application Deployment

  • Overlaps/integrates with CI/CD tools (eg Jenkins)
  • Quite often bespoke software

Application Deployment

  • Fabric
  • Capistrano
  • Ansible
  • GoCD

Orchestration

  • Build the entire stack (networks, routers, security groups, instances etc.)
  • Some tools are cross cloud (ansible, terraform)
  • Higher level than configuration management on individual instances
  • Orchestration of other services (DNS, CDN, LB etc.)

Orchestration

  • AWS CloudFormation
  • OpenStack Heat
  • Ansible
  • Red Hat CloudForms
  • Hashicorp Terraform

Security & Compliance

  • Test Driven Infrastructure
  • Regression testing of security issues
  • PCI or government security standard compliance
  • Security patching

Security & Compliance

  • Chef Inspec
  • Goss (go)
  • testinfra (python)
  • Serverspec (ruby)

Container Orchestration Engines

  • Provide a control plane or operating system on which to run containerised workloads
  • Infrastructure is abstracted so the DevOps team only cares about the application architecture and its scaling requirements
  • Runs on IaaS somewhere

Container Orchestration Engines

"Kubernetes is an open-source system for automating deployment, operations, and scaling of containerized applications."

Container Orchestration Engines

  • Kubernetes
  • Docker Swarm
  • Apache Mesos / DCOS

MTTR vs. MTBF

  • MTTR = Mean Time To Repair
  • MTBF = Mean Time Between Failures
  • MTTD = Mean Time To Detection
  • A low MTTR requires a low MTTD
  • A low MTTR requires Automation
  • A low MTTD requires a good Monitoring
