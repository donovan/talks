System Containers

  • System containers vs. application containers
  • LXC
  • OpenVZ
  • Namespaces and cgroups were written for system containers
  • Unified Kernel API
  • Docker came along and stole the thunder
  • Layers (UnionFS) and workflow

LXC/LXD

  • Written to be a drop-in replacement for KVM
  • 10x increase in number of instances on the same hardware
  • Faster to start all instances
  • Better disk and network IO (up to 40%)
  • Security ...

Summing up

  • Deep integration required particularity for networking and volumes
  • Running containers in VMs is common but not ideal for performance
  • People are running containers in VMs due to security concerns
  • Original container use cases are still valid and interesting (see LXC/LXD)
  • Orthogonal concerns (application vs machine abstractions) can make for difficult fit
  • So many ways to orchestrate things, expect consolidation

Future
